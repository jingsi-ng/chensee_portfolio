Overview

A literary tribute built in pure HTML & CSS

This tribute page was built as a Web Authoring assignment — the brief was to create a multi-page informational website about a subject of our choice using only HTML and CSS. I chose Albert Camus because his work sits at the intersection of literature, philosophy, and politics in a way I find genuinely compelling. The absurd as a design principle, not just a subject.

The site covers Camus's biography through an interactive timeline, his major works, key philosophical themes (The Absurd, Revolt, Living Without Appeal), and an AI-assisted news feed covering contemporary Camus scholarship — built to demonstrate how his ideas remain relevant today.

The constraint was the point: no JavaScript, no frameworks, no libraries. Everything the site does — the timeline, the multi-page navigation, the typographic rhythm — is pure HTML and CSS craft.

Design Decisions

Typography as philosophical argument

The design needed to feel like the subject: serious, intellectual, and European without being cold. The starting point was the hero image — a full-bleed black-and-white photograph of Camus, a man who looked as if he was already living the ideas he would later write down. From that image, the entire palette followed: deep black, off-white, and a warm gold that read as both archival and alive.

Serif and monospace fonts were paired throughout — Times New Roman anchoring the editorial tone, a monospace face used for dates, labels, and cross-references, nodding to the archival quality of historical record. The combination says: this is serious literature, but it's also a system.

Whitespace was the primary layout tool. Camus wrote about silence and the space between things. A dense, busy layout would have contradicted the subject. Every section has room to breathe.

The vertical timeline on the biography page uses a CSS-only approach — a centred line with alternating left/right content blocks — that creates a sense of historical progression without animation or JavaScript. The year markers in gold draw the eye down the page.

Technical Approach

What pure CSS can do

The no-JavaScript constraint turned out to be generative. It forced me to think carefully about what CSS alone is capable of — and to use it properly rather than reach for a script as a shortcut.

The timeline is built with a CSS pseudo-element vertical line (`::before`) and alternating flex-direction on odd/even list items. The expanding year-detail layout uses the sibling combinator to reveal content on `:focus` — no JavaScript required.

The three-column theme cards use CSS Grid with `grid-template-columns: repeat(3, 1fr)` — collapsing to single column on mobile via a single media query breakpoint. The top border accent on each card is a 3px `border-top` in gold, which does significant visual work with almost no code.

Responsive design was treated as a core requirement rather than an afterthought. Every section was tested at 320px, 768px, and 1440px widths. The full-bleed hero image uses `object-fit: cover` with a carefully positioned `object-position` to ensure Camus's face remains in frame at all viewport sizes.

Reflection

What I took away

01 · Learned
Constraints produce clarity. Working without JavaScript forced me to understand CSS layout at a deeper level than I would have otherwise. I came away with a much stronger command of Grid, Flexbox, pseudo-elements, and the cascade — fundamentals that make everything else faster.
02 · Would Do Differently
I'd invest more time in the responsive typography scale. The headings are set in fixed pixel sizes, which works but isn't ideal. I'd use `clamp()` throughout — as I did later in the portfolio — to make the type feel properly fluid across screen sizes rather than just "not broken."
03 · Proud Of
The detail in the timeline. The vertical gold line with alternating content blocks looks simple, but getting the alignment consistent across breakpoints without JavaScript required careful use of negative margins and absolute positioning — the kind of invisible work that only shows if it goes wrong.
