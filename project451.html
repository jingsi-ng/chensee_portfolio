Overview

A dystopian digital archive

Project 451 is a speculative fiction web experience inspired by Fahrenheit 451 — Ray Bradbury's novel about a future where books are banned and "firemen" burn them. The project transforms the novel's themes of censorship, memory, and resistance into an interactive digital archive.

The concept: what if the book-preservers in the novel had a secret digital archive? Users enter a classified, terminal-style interface and navigate through censored documents, hidden records, and narrative-driven content — as if they're accessing something they're not supposed to see.

Design Decisions

Designing a dystopian UI

The entire visual language was designed to feel classified and subversive. Terminal-style monospace typography creates the feeling of command-line interfaces and restricted government systems. Classified modal gateways simulate the experience of accessing forbidden documents.

Colour was used deliberately — high contrast with degraded, worn textures to suggest a world where digital information is both precious and dangerous. CSS animations were custom-built to simulate data transmission, loading states, and system access sequences rather than decorative motion.

Every UI decision served the narrative. The interface itself is part of the story.

Technical Approach

Astro + Supabase architecture

Astro was chosen for its component-based architecture and static site generation (SSG) model — perfect for a content-heavy experience that needed to be fast and deployable. Astro's partial hydration model meant JavaScript was only loaded where it was actually needed, keeping performance tight.

Supabase handled backend database management and dynamic content delivery — allowing the archive content to be stored, queried, and rendered dynamically while keeping the frontend lean. This was my first time integrating a real backend service into a frontend project.

Interactive onboarding logic and state-based transitions were built in vanilla JavaScript — managing which "level" of the archive a user had accessed and what content to reveal next.
