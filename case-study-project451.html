Overview

A dystopian digital archive

Project 451 is a web experience inspired by Fahrenheit 451, a classic novel by Ray Bradbury about a future where books are banned and destroyed. This project translates the novel's themes of censorship and memory preservation into an interactive digital archive.

The core concept asks a simple question: what if the book preservers in the story built a secret digital archive? Users enter a classified computer interface to navigate through hidden records and forbidden texts. The goal was to make visitors feel like they are accessing a highly restricted system.

Design Decisions

Creating an immersive interface

My main goal was to ensure every UI element fit into the fictional world. I did not want to just build a standard website about Fahrenheit 451. I wanted to design a website that looks like it exists inside that dystopian universe.

I chose a dark color palette and monospace typography to build the atmosphere. These visual choices represent an underground system built by people hiding from the authorities. The opening "WARNING: RESTRICTED AREA" modal immediately sets the tone. It asks the user to make an active choice before they can even view the archive.

I also borrowed status indicators from real computer systems, such as SYSTEM ONLINE or ARCHIVE STATUS: SECURE. These familiar institutional elements make the fictional experience feel much more realistic and engaging.

I kept CSS animations simple and intentional. Elements like a blinking cursor and a sequential loading text create a stronger atmosphere than complex scrolling effects.

Technical Approach

Building the frontend and backend

For this project, I wanted to push myself to build a fully functional web application instead of just a static prototype.

I chose Astro as my main framework. Its component-based architecture allowed me to build reusable UI parts easily. At the same time, its static site generation (SSG) features kept the website fast and simple to deploy. Astro's island architecture also ensured that JavaScript was only loaded when absolutely necessary.

For the backend, I integrated Supabase. The preserved texts are stored in a live PostgreSQL database and fetched when the page loads. This was my first time connecting a frontend project to a live database, and it helped me understand how data flows in a real application.

I also implemented user authentication using Supabase Auth. Visitors must register a virtual keycard and log in to access the restricted archive. I managed the session state using vanilla JavaScript to ensure the restricted routes remain secure.

Challenges

Overcoming technical problems

The biggest challenge was connecting Supabase's dynamic database with Astro's static site generation. Since the archive content is user-generated, it cannot be pre-rendered at build time. I solved this by selectively using Astro's server-side rendering mode for the dynamic routes while keeping the rest of the site static.

I also chose to manage the client-side session state using vanilla JavaScript instead of a framework like React. Writing the authentication checks and redirects from scratch was difficult. However, it forced me to deeply understand how web applications handle user sessions behind the scenes.

Finally, getting the warning modal to feel right took time. I had to balance storytelling with usability. If the animations were too long, it would annoy the user. I eventually settled on a simple typewriter reveal effect to keep it engaging but fast.

Reflection

What I learned

01 / Learned
Connecting to a backend changes how you design. Once the archive became a live database rather than static HTML, I had to think about empty states, loading screens, and errors. It taught me that UX design and software engineering are deeply connected.
02 / Would Do Differently
Planning the database structure is crucial. I had to rebuild my Supabase tables halfway through the project because my initial plan did not support data filtering. Next time, I will spend more time designing the data structure before writing any code.
03 / Proud Of
Maintaining a consistent story. I am proud that every part of the website fits the same dystopian theme. From the login screen to the empty search results, the visual design and the code work together to tell a complete story.
