Overview

A personal portfolio disguised as a classified archive

The Curated Archive is a personal portfolio website built with WordPress. The goal of this Web Authoring assignment was to create a functional Content Management System (CMS) to showcase my projects.

Instead of building a standard design portfolio, I wanted to create a cohesive visual narrative. Inspired by the dystopian aesthetic of my previous Fahrenheit 451 project, I positioned myself as a "digital archivist". Every project is presented as a classified document or a preserved file, transforming a simple portfolio into a highly immersive user experience.

Design Decisions

Vintage aesthetics and tactile textures

To bring the archival identity to life, I replaced the pure black backgrounds with a deep forest black to make the site feel warmer and more mysterious. I used cream paper tones for the project cards and applied antique gold accents to mimic vintage stamps and official seals.

Typography was crucial for this concept. I paired Playfair Display for elegant headings with Special Elite for the "classified card" element to create a realistic typewriter aesthetic. I also used Courier Prime for code-like terminal elements.

To make the archival look authentic, I implemented a custom SVG-based noise texture using the CSS `feTurbulence` filter. This generated a subtle film-grain effect across the entire website without relying on heavy image files, which kept the loading times fast.

Technical Approach

Balancing native blocks and custom code

I built the site using the native WordPress Block Editor and Full Site Editing (FSE) capabilities. Initially, I forced myself to use only standard blocks because I wanted to master the native CMS tools. However, I soon realized that complex layouts, like the intricately layered "classified card" on the homepage, required exact precision. I strategically integrated Custom HTML blocks to achieve these specific designs.

I wrote over 1,400 lines of Additional CSS to customize the Twenty Twenty-Five theme. A significant challenge was battling CSS specificity. The default theme styles were very strict, so I had to use highly targeted CSS declarations to ensure my typography choices and custom hover animations applied correctly across all pages.

I also had to modify the global layout constraints. The default content width felt too narrow for a desktop portfolio, so I adjusted the Global Styles layout settings to a maximum width of 1400px, allowing the design to breathe and look professional on large monitors.

Reflection

What I learned

01 / Learned
Knowing when to break the rules. I learned that mastering a CMS is not just about understanding its built-in features. It is equally important to know exactly when the native tools are insufficient and when to step outside those boundaries with custom HTML and CSS.
02 / Would Do Differently
Switch to Custom HTML earlier. I spent hours trying to force native WordPress blocks to create complex, overlapping layouts. Next time, I will accept the limitations of the Block Editor much earlier and use custom code for highly bespoke UI components to save time.
03 / Proud Of
The cohesive storytelling. I am proud that I successfully transformed a standard WordPress theme into a unique storytelling device. From the SVG film grain to the typewriter fonts and the student ID file numbers, every detail supports the illusion of a classified digital archive.
